flutter bloc 模式注意事项

  • State 模型可以不用 const 修饰
  • Bloc、Repository 、 API 的 Model 最好不共用。Bloc 模型添加 fromRepository 构造函数
  • State 中的状态枚举，最好添加 isSuccess 等扩展方法，扩展类追号以 NameStatusX 格式命名
  • API、Repository 中的 Model 参数最好是加上 required。其中 API 不继承 Equatable
  • 如果一个页面的构成是由多个不相干的功能组成，数据也没有关联，State 没有继承关系，就可以考虑使用拆分 State
  • State 模型参数都不加 required
  • State 模型一般包含这几个状态 initial、progress、success、failure 如果是列表则 initial、progress、refreshed、refreshedFailure、loaded、loadedFailure
  • 巧用 identical 方法判断相等。可空与非可空的两个变量可以使用这个方法判断是否相等
